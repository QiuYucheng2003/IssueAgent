一个标准 Agentic Workflow 的通用研发流程
当你自己要去落地一个新的 Agent 项目时，通常是这么做的：

拆任务：把一个大目标（比如写一篇行业研报）拆成 5 个步骤（搜集、提炼、大纲、草稿、校对）。

定角色：为每个步骤编写极其专注的 Prompt，杜绝上下文污染。

连管线：用 LangGraph 这样的框架把这 5 个步骤连成图，加上必要的 Python 工程代码（如去重、文件保存）。

加反馈：在关键卡口加入大模型或人类的评审机制，不合格就打回重写
