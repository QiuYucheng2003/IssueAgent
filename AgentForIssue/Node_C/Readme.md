# Research-SE-Agent: Node C (0-Day GitHub Wild Hunter)

## 📌 简介
本脚本为自动化漏洞挖掘智能体系统的**节点 C（全网零日漏洞巡航猎手）**。本节点实现了真正的**无人值守全网主动扫描**。它自动动态读取节点 B 沉淀的结构化缺陷规则通缉令（`Node_B_rules.json`），联动 **GitHub Code Search API** 在全网盲搜 star 数量较多、热门、有名气的线上开源项目代码切片，并利用大语言模型（LLM）进行云端控制流判决，批量筛选历史上从未被申报过（0-Day）的真实代码误用候选，供后续 Node_D 进一步验证。

## 🚀 核心任务与输出指标
本节点抛弃了多余的报题文本封装，将大模型的推理算力集中于高价值漏洞的定位与根因剖析，核心输出严格对齐以下 4 项指标：
1. **真实未申报缺陷候选 (0-Day Candidate)**：优先从 GitHub 上 star 数量较多、热门、有名气的项目中筛选未被申报过的真实代码误用候选，最终真实性验证交由后续 Node_D 完成。
2. **目标开源仓库名 (Target Repository)**：精准锚定受影响的开源项目。
3. **涉及到文件路径 (File Path & URL)**：提供线上绝对文件路径与代码直达链接。
4. **双语漏洞根因与分类 (Bilingual Root Cause & Typology)**：输出深度中英文控制流漏洞成因推导，并精准定性其缺陷分类（如安全上下文逃逸、内存泄漏、跨任务状态污染等）。

## 📥 输入说明
* **知识配置**：`../Node_B/Node_B_rules.json` (自动向上寻址读取)
* **认证凭证**：系统环境变量 `GITHUB_TOKEN` 与 `DEEPSEEK_API_KEY`

## 📤 产出物说明
* `Node_C_GitHub_0Day_Report.md`：全网主动筛选成果报告。每一个捕获到的候选均包含：开源仓库名、线上源码直达链接、中英文双语控制流漏洞根因推导、以及缺陷类型分类，后续可交由 Node_D 做进一步验证。
